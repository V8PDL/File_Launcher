Внешние компоненты. 

Программа запоминает и ведет учет двум категориям:
1. Файлы
Файлы описываются списком в самой программе и содержатся в оперативной памяти для быстродействия. 
При этом также ведется запись логов в файл Files List.txt , при желании его можно режактировать вручную, внося какие-то необходимые изменения. 
Каждая строка соответствует файлы, в форме Имя_Файла|Название_В_Программе|Группа
Это позволяет сразу же иметь набор файлов, оставленный после предыдущей работы. 
2. Группы/типы
Группы - некоторые наборы названий, чем-то объединяющих расширения файлов. Изначально приведено немного, но по необходимости их можно добавлять как напрямую в файл лога - Types Dictionary.txt , так и через сам интерфейс. 
Синтаксис: Имя_Типа: _Перечисление_относящихся_к_нему_расширений_через_пробел, например
image: .jpg .png .bmp

ВАЖНО. Названия типов и имен файлов в программе не должны включать в себя ":" и "|"

Интерфейс.

1. Кнопка "Select file" 
Позволяет выбрать файл с помощью стандартных средств Windiws. 
Результат - полное имя файла - записывается в редактируемом поле справа. 
Особенности Windows не позволяют так же просто реализовать выбор директории (вместо файла) в стандартном диалоговом окне. 
Чтобы работать с папками, необходимо поставить галочку в чекбоксе "Directory" и ввести путь самостоятельно. Проще это сделать, выбрав путь к некоторому файлу в этой папке, а затем удалив его название. 
Программа не может читать системные папки и файлы, не будучи запущенной от администратора и в принципе не была тестирована в этом ключе. 
Возможно множество необработанных ошибок, ведущих к различным последствиям.

2. Кнопка "Reload directory" 
Считывает лог файла групп, чтобы использовать в программе. Необходимо использовать, если какие-либо изменения были внесены вручную. 
Возможность заново прочитать лог используемых фалов, на мой взгляд, используется реже, но при необходимости достаточно будет перезапустить программу. 

3. Выпадающий список справа от надписи "Groups"
Позволяет выбрать группу, чтобы фильтровать поиск при галочке в чекбоксе "Directory". 
При использовании с одним файлом принудительно задает ему эту группу, даже если файлы такого расширения относятся к другому типу. При этом добавляется новая запись с объявленной группой, без параметров.
Может оставаться пустым, в таком случае группа будет определена самостоятельно. 
Если расширение не относится ни к одной из известных групп, то файл будет отнесен в "Other"

4. Редактируемое поле слева от "Name"

Задает представляемое имя файла в программе, может абсолютно не зависеть от названия в файловой системе, но по умолчанию берется на его основе.
Может быть не указано.

5. Кнопка "Add file"
Загружает файл в программу исходя из указанного пути, устанавливает ему и тип, если не заданы. 
Может осуществлять поиск по папкам на основе их расширений. 
Расширения берутся из ключевого слова, но также могут быть заданы в редактируемом поле	ниже, под текстом.
Во избежание больших задержек и зависания приложения при больших объёмах данных, поиск файлов в каталогах производится в фоне. 
Это может вызвать непредвиденные последствия при попытке дополнительно взаимодействовать с изменяемыми фалйами, кроме того, кроме того, возможны непрогнозируемые вылеты при столкновении с файлами и папками с ограниченным доступом.
Этот режим в целом безопасен, но на терабайтных хранилищах его лучше не использовать, с выпадающим списком на 50.000 элементов WPF сам начинает тормозить. 
Возможно, для появления найденных файлов в выпадающем списке File будет необходимо добавить единичный файл самостоятельно. 

6. Кнопка "Add group"
Добавляет новую группу, заданную в поле напротив "Group", имеющую расширения, указанные в поле ниже. 
На самом деле, это не вполне расширения - скорее окончания файлов. 
Таким образом при необходимости можно искать, скажем, картинки по дате, если это позволяет формат, задав в "расширениях" 27112020.jpg 

7. Выпадающий список "File"
Необходим для выбора файла. Отображается его название, однако при выборе в поля ниже вносятся другие его параметры - имя, название и тип. 

8. Кнопка "Delete file (from list only!) удаляет выбранный файл из коллекции. Он всё еще доступен в файловой системе и может быть загружен заново. 

9. Кнопка "Delete file (at all!)" 
Удаляет файл с компьютера. Восстановить его можно будет только специальным ПО. 

10. Чекбокс "Full path" 
Определяет, выводится ли для выбранного файла полное имя, или же только название. Изменив путь целиком, можно перемещать файл, одновременно переименовывая. 

11. Поля "Name", "File name" опеделяют имя файла в программе и в файловой системе. 

12. Выпадающий список "Group"
Идентичен тому, что выше, но не может изменяться. 
Выбор элемента влечет за собой заполнение его параметрами поля ниже.

13. Кнопка "Change file" 
Меняет свойства файла в программе на выбранные пользователем На реальном файле отражается только путь. 

14. Кнопка "Open file" 
Открывает файл средством по умолчанию для этого расширения. 

15. Поле "Extensions" внизу
Принимает значения параметров группы - точно так же через пробел, с точкой. 

16. Кнопка "Change group"
Меняет расширения, принадлежащие группе, на заданные пользователем.

17. Кнопка "Delete group" 
Удалят группу. Все файлы, принадлежавшие к ней, автоматически меняют группу на "Other".
